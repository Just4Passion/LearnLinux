## 内核内存分配
1. **页分配器**: 分配物理页框，是最底层的分配接口
    - alloc_pages(gfp_mask, order): 分配2^order个连续页框
    - alloc_page(gfp_mask): 分配单个页框（order=0）
2. **SLAB分配器**: 分配小内存块
    - kmalloc(size, flags): 分配指定大小的物理连续内存
    - kzalloc(size, flags): 分配并清零
    - kcalloc(n, size, flags): 分配数组并清零
    - krealloc(p, size, flags): 重新调整内存大小
    - kmem_cache_create() / kmem_cache_alloc(): 专用缓存分配
3. **vmalloc区域**
    - vmalloc(size): 分配虚拟连续内存
    - vzalloc(size): 分配并清零
    - vmalloc_user(size): 为用户空间分配
    - vfree(addr): 释放vmalloc分配的内存
4. **DMA内存**
    - dma_alloc_coherent(): 分配DMA一致内存

## ioctl()命令
1. **格式**: 设备类型(8-bit) + 序列号(8-bint) + 方向(2-bit) + 数据尺寸(13/14-bit)
    - 设备类型
        - 自定义值
        - 0-0xff, ioctl-number.txt给出了一些推荐和已经被使用的值
        - 新定义的值避免冲突
    - 命令码序列号
    - 方向
        - _IOC_NONE: 无数据传输
        - _IOC_READ: 读取
        - _IOC_WRITE: 写
        - _IOC_READ|_IOC_WRITE: 双向
    - 命令码的数据长度
        - 标识涉及的用户数据的大小
2. **命令定义宏**
    - 命令宏
        - _IO()
        - _IOR()
        - _IOW()
        - _IOWR()
        - _IOC()
    - 参数
        - type: 设备类型字段
        - nr: 序列号字段
        - size: 数据长度字段

## mknod命令
1. **作用**
    - 用于创建特殊文件. 
    - 特殊文件作为内核与用户空间交互的接口, 当用户读写这些文件时, 内核会根据设备号调用响应的驱动程序
2. **命令格式**: mknod [选项] 名称 类型 [主设备号 次设备号]
    - 类型: b(块设备), c(字符设备), p(FIFO管道)
3. **使用场景**
    - 当你在内核创建了一个字符设备, 但是设备没有绑定device时, 可以通过mknod来为设备创建一个设备节点
    - 然后通过访问设备节点来访问字符设备驱动

## 编译乱序控制: 编译器屏障
```
#define barrier() __asm__ __volatile__("" ::: "memory");
```
1. **__volatile__**: 禁止编译器优化掉这段汇编
2. **("" ::: "memory")**
    - "": 空汇编指令
    - ::: : 无输入操作数, 无输出操作数
    - "memory": 告诉编译器, 内存可能被修改

2. **屏障指令**
    - 屏障指令
        - DMB(数据内存屏障): 在DMB之后的显式内存访问执行前, 保证所有在DMB指令之前的内存访问完成
        - DSB(数据同步屏障): 等待所有在DSB指令之前完成的指令完成
        - ISB(指令同步屏障): Flush流水线, 使得所有ISB之后执行的指令都是从缓存或内存中获取的
    - 读写屏障
        - mb()
        - rmb()
        - wmb()
    - 寄存器读写屏障: 外设可能需要固定的寄存器读写顺序, 需要等某些寄存器配置完成才能配置其他寄存器, 这个时候就需要屏障刷新
        - __iormb(): 
        - __iowmb(): 
    - 读写寄存器接口: 区别在于有无屏障
        - readl_relaxed()
        - readl()
        - writel_relaxed()
        - writel()
    - 自旋锁, 互斥体等互斥逻辑依赖屏障指令

4. **并发控制机制**: 共享资源访问
    - 中断屏蔽
        - OS依赖中断, 单核屏蔽中断方式效用太低
        - 多核仅屏蔽一个CPU不行
    - 原子操作: 通过LDREX, STREX实现. 与CPU架构强相关
        - 原子整型操作
        - 原子位操作
    - 自旋锁(spin lock)
        - 原理: 
            - CPU先执行一个原子操作，该操作测试并设置(Test-And-Set)某个内存变量
            - 如果测试结果表明锁被占用, 则程序将在一个小的循环内重复这个"测试并设置"操作, 即进行"自旋"
            - 如果测试结果表明锁已空闲, 则程序获得这个自旋锁, 并继续执行
        - 自旋锁
        - 读写自旋锁
        - 顺序锁
        - RCU: Read-Copy-Update
    - 信号量
    - 互斥体
        - 存在阻塞, 临界区大, 非中断情况下使用
