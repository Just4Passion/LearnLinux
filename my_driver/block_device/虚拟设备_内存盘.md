# 虚拟机查看块设备
```
ls /sys/block/
dm-0  fd0  loop0  loop1  loop2  loop3  loop4  loop5  loop6  loop7  sda  sr0
```
1. **总结**
    - 都是虚拟设备
    - dm-0: 
    - fd: 
    - loop:
    - sda:
    - sr0: 
2. **sda**: 系统主磁盘
    - 类型: SCSI磁盘设备
    - 接口： 
        - 物理/虚拟接口: SATA, SCSI
        - 逻辑接口: 通常抽象为SCSI子系统下的设备
    - 命令
        - ls -l /sys/block/sda: 查看设备链接，可以看到它在PCI总线上的位置
        - cat /sys/block/sda/device/type: 输出0, 0代表磁盘
3. **sr0**: 光盘驱动
    - 类型： SCSI CD-ROM/DVD-ROM设备
    - 接口: 通常被虚拟化为SATA或SCSI光驱
    - 协议: ATAPI(ATA数据包接口)或MMC
    - 命令
        - ls -l /sys/block/sr0: 
        - cat /sys/block/sr0/device/type: 输出5, 5代表CD/DVD
4. **dm-0**: Device Mapper设备
    - 类型: 内核Device Mapper框架创建的虚拟块设备. 不是物理设备, 是在现有块基础上构建的逻辑层
    - 接口: Device Mapper框架(通过ioctl系统调用控制)
    - 协议: 无特定硬件协议
    - 用途
        - LVM2逻辑卷: dm-0 很可能是系统的根文件系统或交换分区所在的逻辑卷
        - LUKS加密磁盘: 
    - 命令
        - sudo dmsetup info dm-0: 查看dm-0的名称和类型
        - ls -la /sys/block/dm-0/slaves/: 查看依赖的底层设备. sd3, 表示建立在/dev/sd3分区上
5. **loop0到loop7**：回环设备
    - 类型: 虚拟块设备, 用于将一个普通文件映射为一个块设备来访问
    - 接口: 回环设备驱动(loop), 通过losetup命令控制
    - 协议: 无硬件协议。它将文件系统层的读写操作转换为对底层文件的读写操作
    - 用途
        - 挂载ISO镜像: Ubuntu 的某些组件（如 snap 包）大量使用回环设备来挂载 squashfs 镜像
        - 挂载磁盘镜像: 
    - 命令
        - losetup /dev/loop0
6. **fd0**: 软盘驱动
    - 类型：软盘设备
    - 接口: 传统PC的软盘控制器接口
    - 协议: 软盘控制器专用命令



# 开发板查看块设备
```
ls /sys/block/
dm-0  dm-5    loop12  loop3  loop8         ram0   ram13  ram4  ram9
dm-1  loop0   loop13  loop4  loop9         ram1   ram14  ram5  zram0
dm-2  loop1   loop14  loop5  mmcblk2       ram10  ram15  ram6
dm-3  loop10  loop15  loop6  mmcblk2boot0  ram11  ram2   ram7
dm-4  loop11  loop2   loop7  mmcblk2boot1  ram12  ram3   ram8
```
1. **总结**
    - dm:
        - 逻辑卷设备: Device Mapper 设备
        - 常见用途
            - Android/Linux 动态分区：Android 或类似的系统，其中 system, vendor, product 等分区是通过 Device Mapper 从 super 物理分区中动态映射出来的
            - 加密：如果启用了磁盘加密
    - loop:
        - 虚拟块设备，用于将文件映射为块设备
        - 常见用途：在嵌入式开发中，常用于挂载系统镜像或 snap 应用
    - mmcblk:
        - MMC块设备, 板载eMMC芯片
        - mmcblk2: 主用户分区, 文件系统、应用程序数据存放的地方. 
        - mmcblk2boot0和mmcblk2boot1: 两个引导分区, SoC通常从这两个分区之一启动
    - ram: 
        - 内存模拟磁盘
        - 常见用途:
            - 作为超高速的临时存储
            - 内核开发和块层测试
            - 挂载需要基地延迟的文件系统
    - zram:
        - 压缩内存块设备
        - 接口与协议: 内核的zram模块, 数据在写入时会被在线压缩, 读出时解压
        - 常见用途
            - 用作交换空间: 
            - 用作通用压缩磁盘: /tmp目录
