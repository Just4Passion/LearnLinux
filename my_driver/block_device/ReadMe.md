4. **高级块设备框架**: 中间层
    - 本身不驱动硬件, 通过对现有设备进行变换、组合或虚拟化, 创建出新的、功能更强大的块设备
    - Device Mapper: 设备映射器
        - 功能: 将一个或多个底层块设备, 通过特定的映射规则, 组合成一个新的逻辑块设备
        - 工作原理
            - 接收来自通用块层的bio请求
            - 根据其"映射表", 修改bio的目标扇区, 并将其转发给对应的底层设备
            - 这种映射可以是1:1、1:N、N:1各种映射关系
        - 典型应用
            - LVM2: 逻辑卷管理，实现动态扩容、快照
            - dm-crypt: 全盘加密
            - dm-multipath: 多路径I/O, 提供冗余和负载均衡
            - dm-thin: 精确配置
        - driver/md/dm*.c
    - MD: Multiple Device
        - 功能: 对接多个物理块设备
        - 工作原理
            - MD是Linux的软件RAID实现
            - 它将多个物理磁盘组成一个逻辑的MD设备, 并根据配置的RAID级别, 对数据进行条带化、镜像或校验计算, 然后将I/O请求分发到成员盘上
        - driver/md/md.c
    - 虚拟, 网络块设备
        - Loopback Device: 回环设备
            - 功能: 对接一个普通文件
            - 工作原理
                - 将一个文件虚拟成一个块设备
                - 当对这个块设备进行 I/O 操作时，Loopback 驱动会将其转换为对底层文件的读写操作。这是挂载 .iso 镜像文件的基础
            - drivers/block/loop.c
        - NBD: Network Block Device
            - 功能: 远程网络存储服务器
            - 工作原理
                - 将一个块设备与网络协议栈连接起来
                - NBD 客户端驱动接收块层请求，将其封装成网络报文发送给 NBD 服务端；服务端在真实的块设备上执行操作并返回结果
            - drivers/block/nbd.c

# 存储设备

## 存储介质
1. **Flash**
    - 是一种非易失性存储介质，断电后数据不丢失
    - 从 EEPROM 发展而来，按内部存储结构不同，分为 NOR Flash 和 NAND Flash 两大流派
2. **Nor Flash**
    - 特点: 
        - 共享I/O线: 地址和数据共用一组I/O线分时复用, 无法直接随机访问, 只能按页读取
        - 读、写、擦除速度快
        - 容量巨大, 成本极低
    - 缺点: 存在坏块, 需要复杂的坏块管理、磨损均衡、ECC纠错管理算法
    - 典型应用
        - 是所有大容量存储的基石：SSD、U盘、SD卡、eMMC、UFS、手机/平板存储, 都是 NAND Flash 颗粒
3. **Nor Flash**
    - 特点
        - 独立地址线和数据线. 读速度快
        - 芯片内执行
    - 缺点: 写和擦除速度慢, 容量小
    - 典型应用
        - 存储Bootloader, 固件, 操作系统内核等需要直接运行或快速启动的代码

## 存储设备: 从裸介质到可管理的设备
1. **无智能控制器的SPI接口方案**
    - 在NOR或NAND Flash颗粒上加上一个简单的SPI接口控制器, 就成了SPI Flash
    - SPI Nor Flash
    - SPI Nand Flash    
        - 需要主机自己处理坏块管理、ECC等
2. **带智能控制器的复杂方案**
    - NAND Flash颗粒 + 智能控制器 + 标准接口
    - 典型
        - SD卡, eMMC, UFS, SSD

## 存储设备接口和协议
    - 物理接口/总线: 负责定义物理连接、电气特性、底层信号传输方式
    - 上层协议: 负责定义数据包的格式、命令集、队列管理、错误处理等高级功能
    - SCSI协议
        - SCSI/SAS/SATA/UFS/ACHI
    - PCIe接口 + NVMe协议
    - SATA接口 + AHCI协议
    - SAS接口 + SCSI协议
    - M-PHY接口 + UFS协议
    - MMC总线 + eMMC协议
    - SD总线 + SD协议

### PCIe接口 + NVMe协议: 峰值16GB/s
1. **PCIe**
    - 物理层
        - 采用高速串行差分信号
        - 使用多个Lane进行数据传输. 常见有x1, x4, x8, x16等配置
    - 拓扑结构
        - 点对点连接
        - 每个设备直接与CPU或PCIe交换芯片的端口相连, 独享带宽
    - 核心优势: 极高的带宽（PCIe 5.0 x4 可达 ~16 GB/s）、极低的延迟（直连 CPU）、支持多通道并行传输
2. **NVMe协议**
    - 设计目标: 专为非易失性存储器（NAND Flash 等）优化，充分发挥 PCIe 的高并行、低延迟特性
    - 核心机制
        - 多队列架构：支持多达 65535 个 I/O 队列，每个队列深度可达 65536. 允许CPU多核并行提交和完成I/O请求
        - 精简命令集: 相比于AHCI, NVMe的命令处理路径极短, 大幅度降低了CPU开销
        - 主机内存缓冲: 允许SSD直接访问主机内存中的数据结构, 减少数据拷贝
3. **Linux驱动**
    - PCIe驱动：负责枚举设备, 配置空间, 管理MSI-X中断
    - NVMe驱动: 利用 PCIe 提供的地址空间和中断，实现 NVMe 协议栈，最终将设备注册为一个标准的块设备

### SATA接口 + AHCI协议: 峰值550MB/s
1. **SATA接口**
    - 物理层
        - 采用串行差分信号, 但是只有一个通道
    - 拓扑结构
        - 点对点连接, 一个端口只能连接一个设备
    - 核心特点: 成本低、兼容性好、计数成熟
2. **AHCI协议**
    - 设计目标: 为 SATA 硬盘设计，继承了传统机械硬盘的编程模型
    - 核心机制
        - 单命令队列: 最多支持 1 个命令队列，深度为 32。所有 I/O 请求必须在这个单一队列中排队，多核 CPU 访问时会产生锁竞争
        - 寄存器交互: 命令提交需要多次读写特定的寄存器，导致较高的 CPU 开销和延迟
        - 支持NCQ: 允许硬盘对队列中的命令进行重排序，以优化磁头寻道（对机械硬盘有效，对 SSD 意义不大）
3. **Linux驱动**
    - SATA控制器驱动: 通常基于libata库, 位于drivers/ata/目录下
    - AHCI驱动: drivers/ata/ahci.c 是通用的 AHCI 控制器驱动。它将 SATA 设备通过 SCSI 子系统中转，最终也注册为块设备


### SAS接口 + SCSI协议: SATA的"企业加强版"
1. **SAS接口**
    - 物理层
        - 与SATA兼容, 但电气特性和协议更强大
    - 拓扑结构
        - 支持交换式架构，**一个 SAS 域可以通过扩展器连接最多数万个设备**，这是其核心优势
    - 核心特点: 高可靠性、长距离传输、双端口冗余（关键特性，防止单点故障）
2. **SCSI协议**
    - 设计目标: 一个历史悠久且功能强大的通用命令协议，不仅用于存储，也用于扫描仪、打印机等
    - 核心机制
        - 丰富的命令集：除了基本的读写，还有大量管理和诊断命令
        - 命令队列：支持 TCQ，允许设备优化命令执行顺序
        - 逻辑单元号：一个 SCSI 设备可以包含多个逻辑单元，每个单元独立管理
3. **Linux驱动**
    - SAS 驱动：通常由 mpt3sas, hpsa 等驱动管理
    - SCSI 子系统：位于 drivers/scsi/，它是 Linux 存储栈的核心部分。所有 SCSI/SAS/SATA/UFS 设备最终都通过 SCSI 子系统处理，生成块设备


### M-PHY接口 + UFS协议: 峰值4.6GB/s
1. **MIPI M-PHY**: 
    - 物理层
        - MIPI联盟定义的串行物理层标准, 专为移动设备优化. 特点是功耗低、高带宽、抗电磁干扰
        - 链路架构
            - Lane: 一对基础的差分信号线
            - Sub-Link: 由一个或一组Lane构成. 一个Sub-Link内的所有Lane共享同一个时钟源
            - Link: 一个完整的双向通信链路, 由两个Sub-Link组成, 一个用于发送, 一个用于接收
        - 两种Lane类型
            - Type-I Module: 时分双工. 同一对差分线既用于发送也用于接收，通过时间片切换方向
            - Type-II Module: 频分双工. 两对差分线，一对固定用于发送，另一对固定用于接收
    - 拓扑结构
        - 点对点连接
    - 核心特点
        - 支持多种高速Gear(速度等级), 功耗管理极其精细, 可在极低功耗状态和高速传输状态间快速切换
            - v3.0	HS-Gear3	5.8 Gbps	8b/10b 编码	               UFS 2.0/2.1
            - v4.0	HS-Gear4	11.6 Gbps	8b/10b 编码	               UFS 3.0/3.1
            - v4.1	HS-Gear5	23.32 Gbps	自适应均衡	                UFS 4.0/4.1
            - v5.0	HS-Gear5	23.32 Gbps	多电平信号，优化休眠	     UFS 4.1
            - v6.0	HS-Gear6	46.6 Gbps	PAM4 调制，1b1b 编码	    UFS 5.0
        - 极简信号: 一个M-PHY链路最少只需要2根差分信号线
        - 双模式运作: HS和LS
2. **UFS协议**
    - 设计目标: 为移动设备量身打造, 继承并超越了SCSI和NVMe的许多思想
    - 核心机制
        - 全双工传输: UFS可以同时进行读写操作
        - 命令队列: 支持类似NVMe的多队列和命令排序, 大幅提升多任务性能
        - SCSI命令集: 基于简化的SCSI命令模型, 功能强大且成熟
3. **Linux驱动**
    - UFS驱动
        - 位于 drivers/ufs/ 目录，它直接管理 M-PHY 接口，并通过 UFS 协议层与设备通信。最终也通过 SCSI 子系统或直接注册为块设备

### MMC总线 + eMMC协议
1. **MMC总线**
    - 物理层
        - 并行总线，有 1/4/8 位数据线，时钟线，命令线。结构简单
    - 拓扑结构
        - 主从模式，一个主机可连接多个从设备（但实践中通常只连一个 eMMC 芯片）
    - 核心特点: 成本极低，引脚少，适合嵌入式设计
2. **eMMC协议**
    - 设计目标：将 NAND Flash 控制器和 Flash 颗粒封装在一个 BGA 芯片中，对外提供标准的 MMC 接口
    - 核心机制：
        - 半双工传输：同一时刻只能进行读或写操作。
        - 无命令队列：必须等待当前命令完成后才能发送下一条命令，无法并行处理。
        - 管理简化：内部控制器处理了坏块管理、磨损均衡等，主机端无需关心
3. **Linux驱动**
    - MMC 子系统
        - 位于 drivers/mmc/ 目录，分为核心层、主机控制器驱动和卡驱动。eMMC 设备会被识别为 MMC 卡，并注册为块设备（如 /dev/mmcblk0）


### SD总线 + SD协议
1. **SD总线**
    - 物理层
        - 与 MMC 总线高度相似，也是并行总线，有 1/4 位数据线，时钟，命令线。物理上兼容性高（通过适配器）。
    - 拓扑结构
        - 主从模式，通常一个卡槽连接一张卡。
    - 核心特点：可插拔、便携、标准化
2. **SD协议**
    - 设计目标：与 eMMC 协议同源，都是基于 MMC 标准发展而来。
    - 核心机制
        - 半双工
        - 无命令队列
        - 与 eMMC 类似。增加了版权保护相关功能。
    - 速度等级：定义了 Class、UHS、Video Speed Class 等速度等级，确保满足不同应用（如视频录制）的写入速度要求
3. **Linux驱动**
    - MMC子系统：与 eMMC 共用同一套驱动框架，SD 卡会被识别为 SD 卡类型，并注册为块设备（如 /dev/mmcblk1）



## 存储设备类型
    - SD卡, EMMC, NAND Flash, Nor Flash, SPI Flash, 机械硬盘, 固态硬盘
    - NVMe(Non-Volatile Memory Express, 非易失性内存主机控制器接口规范)
        - 专为基于闪存的存储设备(如SSD)设计的、基于PCIe总线的高速、低延迟存储协议
        - AHCI（SATA SSD用的协议）的继任者
    - MMC/SD子系统
        - 物理组成
            - 主控制器: Host Controller
            - MMC/SD设备卡: eMMC芯片, SD卡, TF卡. 它们内部集成了Flash存储单元和一个智能控制器, 负责管理闪存和响应Host命令
            - Bus(总线): 连接Host和Card的物理通道, 包括时钟、命令和数据线
        - 架构: 文件系统 -> MMC/SD card层 -> MMC/SD core层 -> MMC/SD host层
        -
